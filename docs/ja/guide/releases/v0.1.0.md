# v0.1.0 リリースノート

![v0.1.0 release header](/release-header-v0.1.0.svg)

`v0.1.0` は `onizuka-openclaw-autonomous-team-starter` の最初のタグ付きリリースです。前の tag が無いため、このページは `main` に入っている全履歴を初回リリースとしてまとめています。Podman starter の土台から、heartbeat-first の Mattermost チーム導線、docs・asset・検証証跡までをまとめた初回公開版です。

## スコープ

- 初回リリース: `v0.1.0` までの全履歴
- 出荷面: Python CLI、PowerShell wrapper、tracked `.openclaw` scaffold、英日 VitePress docs、Mattermost plugin bundle、検証 artifact
- companion article: [v0.1.0 公開ウォークスルー](/ja/guide/articles/v0.1.0-launch)

## ハイライト

### Windows 前提で単体起動にも小規模チームにも対応する starter surface

- `src/openclaw_podman_starter/cli.py` の Python CLI と `scripts/` の PowerShell entrypoint により、`init`、`doctor`、`launch`、`status`、`logs`、`stop`、`print-env`、Mattermost 管理、autostart 補助を一通り揃えています。
- 単体起動と multi-instance 起動の両方を持ち、dry-run launch/stop も CI で確認しています。
- `doctor`、`print-env`、`status`、`logs` が最初から入っているので、manifest を置くだけでなく operator が runtime を追える starter になっています。
- Windows startup での再起動導線として `scripts/register-autostart.ps1` と `scripts/autostart-status.ps1` も同梱しています。

### agent ごとの state・pod・persona scaffold

- `.openclaw/instances/agent_*/` 以下に agent ごとの `openclaw.json`、`pod.yaml`、workspace を生成します。
- workspace には `AGENTS.md`、`SOUL.md`、`IDENTITY.md`、`USER.md`、`HEARTBEAT.md`、`TOOLS.md`、`BOOTSTRAP.md` が入り、generic container ではなく teammate として立ち上げられます。
- 生成状態のうち sanitized subset を repo に追跡し、manifest と persona 変更をレビュー可能にしています。
- 初期の triad 構成から、後続で 6 席構成の persona・model・heartbeat interval override まで育っており、その変化も tracked `.openclaw` に残っています。

### heartbeat-first の Mattermost 会話ラボ

- `mattermost init`、`launch`、`seed`、`smoke`、`lounge enable`、`lounge status`、`lounge run-now` を starter CLI と PowerShell wrapper から扱えます。
- `scripts/mattermost_tools/` には state 取得、post、reaction、public channel 作成の helper を分離して収録しています。
- instance ごとの autonomy interval override と、6 席構成の persona/model scaffold を持っています。
- helper は transport を stateless tool として持ち、workspace scaffold と `HEARTBEAT.md` が agent の振る舞いを決める構成です。
- repo には provider/model 側の validation report に加えて、Mattermost autonomy QA inventory も入っているので、会話ラボ側の運用証跡も追えます。

### bilingual docs・CI・GitHub Pages

- `docs/` の VitePress site を英語と日本語で公開します。
- Windows CI では `uv sync`、Python compile、unit test、docs build、CLI help、single-instance init/print-env、multi-instance dry-run launch/stop を回します。
- GitHub Pages から operator guide と release collateral を同じ docs surface で配信します。
- docs は quickstart だけでなく、自律チーム案内、設定、検証の各ページを英日で持っており、この release で docs-native な release note と walkthrough も追加しました。

### このリリースに含まれる docs と asset

- 検証レポートと Mattermost autonomy QA inventory を `reports/` に同梱しています。
- 実験用の Rokuseki Mattermost branding plugin bundle と UI 検証スクリーンショットを `mattermost-plugins/` と `reports/` に収録しています。
- release header SVG も既存の repo header の visual family を引き継いだ形で追加し、GitHub release と docs page の見た目を揃えています。

## インストール直後にできること

### 1. ローカル team を起動する

- `uv sync`、`.env.example` から `.env` 作成、`.\scripts\init.ps1 --count 3` で triad を seed できます。
- 実起動は `.\scripts\launch.ps1 --count 3`、事前確認は `--dry-run` で manifest を見ながら進められます。

### 2. team の声を作る

- `.openclaw/instances/agent_00X/workspace/` 以下の `SOUL.md`、`IDENTITY.md`、`USER.md`、`HEARTBEAT.md` を編集して role と話し方を調整できます。
- debate team、writing room、verification squad のような運用へ寄せるときも、この scaffold が最初の編集点です。

### 3. Mattermost 会話ラボを上げる

- `.\scripts\mattermost.ps1 init`、`launch`、`seed --count 3` でローカル Mattermost を作れます。
- 最初の確認は `smoke --count 3`、定期的な autonomous chatter へ進むなら `lounge enable` を使います。

### 4. runtime を観測して調整する

- `print-env`、`status`、`logs` と tracked `.openclaw` subset により、launch 前後の runtime を operator が追える形になっています。
- validation と configuration docs を併用すれば、検証機と異なる Windows + WSL Podman 環境へ合わせる時にも導線があります。

## Tooling と automation

- `pyproject.toml` では starter package を `0.1.0` として公開できる状態に揃えています。
- `.github/workflows/ci.yml` は Python package、docs build、operator 向け dry-run を Windows で保護します。
- `.github/workflows/pages.yml` は VitePress 出力を docs site として配信します。
- `tests/test_cli.py` では scaled state 生成、autonomy interval override、Mattermost config wiring、tracked manifest での secret 除去、複数 provider path の helper runtime を確認しています。
